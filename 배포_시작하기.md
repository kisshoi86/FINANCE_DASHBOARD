# 🚀 Finance Dashboard 배포 시작하기

이 문서는 finance_dashboard를 배포하는 가장 간단한 방법을 안내합니다.

## 방법 1: 자동 배포 스크립트 사용 (권장)

### 1단계: PowerShell에서 스크립트 실행

finance_dashboard 폴더에서 PowerShell을 열고:

```powershell
.\deploy_to_github.ps1
```

스크립트가 다음을 자동으로 수행합니다:
- Git 저장소 초기화 확인
- 원격 저장소 설정 안내
- 파일 추가 및 커밋
- GitHub에 푸시

### 2단계: GitHub 저장소 생성 (아직 안 했다면)

스크립트 실행 중 저장소 URL을 입력하라는 메시지가 나오면:

1. https://github.com 접속
2. 로그인 (kisshoi86 계정)
3. 우측 상단 `+` 버튼 > "New repository"
4. 저장소 이름: `FINANCE_DASHBOARD`
5. Public 또는 Private 선택
6. **아래 옵션들은 체크하지 마세요:**
   - ❌ Add a README file
   - ❌ Add .gitignore
   - ❌ Choose a license
7. "Create repository" 클릭
8. 생성된 저장소의 URL을 복사 (예: `https://github.com/kisshoi86/FINANCE_DASHBOARD.git`)
9. 스크립트에 URL 입력

### 3단계: Streamlit Cloud 배포

코드가 GitHub에 푸시되면:

1. https://share.streamlit.io 접속
2. GitHub 계정으로 로그인
3. "New app" 클릭
4. 설정:
   - **Repository**: `kisshoi86/FINANCE_DASHBOARD`
   - **Branch**: `main`
   - **Main file path**: `dashboard.py`
5. "Advanced settings" 클릭
6. "Secrets" 섹션에 다음 환경 변수 추가:

```
SNOWFLAKE_ACCOUNT=cixxjbf-wp67697
SNOWFLAKE_USER=songahreum
SNOWFLAKE_PASSWORD=Fnfsnowflake2025!
SNOWFLAKE_WAREHOUSE=DEV_WH
SNOWFLAKE_DATABASE=FNF
SNOWFLAKE_SCHEMA=SAP_FNF
SNOWFLAKE_ROLE=PU_SQL_SAP
```

7. "Deploy!" 클릭
8. 배포 완료 후 제공되는 URL로 접속

## 방법 2: 수동 배포

스크립트를 사용하지 않으려면 다음 명령어를 직접 실행하세요:

### 1. Git 저장소 초기화

```powershell
# finance_dashboard 폴더에서 실행
git init
git branch -M main
```

### 2. 파일 추가 및 커밋

```powershell
git add .
git commit -m "Initial commit: Finance Dashboard"
```

### 3. GitHub 저장소 생성 및 연결

먼저 GitHub에서 저장소를 생성한 후:

```powershell
git remote add origin https://github.com/kisshoi86/FINANCE_DASHBOARD.git
git push -u origin main
```

### 4. Streamlit Cloud 배포

방법 1의 3단계를 참고하세요.

## ⚠️ 주의사항

### 경로 문제

PowerShell에서 한글 경로로 이동할 때 문제가 발생할 수 있습니다. 다음을 시도해보세요:

```powershell
# 방법 1: 짧은 경로 사용 (PowerShell에서 확인)
(Get-Item "C:\Users\AC1160\OneDrive - F&F\바탕 화면\finance_dashboard").FullName

# 방법 2: 탐색기에서 폴더 열고 PowerShell 실행
# 폴더에서 우클릭 > "PowerShell에서 열기"
```

### Git 저장소 위치 확인

현재 Git 저장소가 올바른 위치에 있는지 확인:

```powershell
git rev-parse --show-toplevel
```

finance_dashboard 폴더 경로가 나와야 합니다.

### 인증 문제

GitHub에 푸시할 때 인증이 필요할 수 있습니다:

1. Personal Access Token 생성:
   - GitHub > Settings > Developer settings > Personal access tokens > Tokens (classic)
   - "Generate new token" 클릭
   - `repo` 권한 선택
   - 토큰 생성 후 복사

2. 푸시 시 사용자 이름과 비밀번호 대신 토큰 사용:
   - Username: GitHub 사용자 이름
   - Password: 생성한 Personal Access Token
