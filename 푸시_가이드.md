# GitHub 푸시 가이드

finance_dashboard 폴더에서 GitHub에 코드를 푸시하는 방법입니다.

## 방법 1: 배치 파일 사용 (가장 간단)

1. **finance_dashboard 폴더에서 `git_push.bat` 파일을 더블클릭**하거나

2. PowerShell에서 실행:
   ```powershell
   .\git_push.bat
   ```

배치 파일이 자동으로:
- Git 저장소 초기화 (필요시)
- 원격 저장소 연결 확인/설정
- 파일 추가 및 커밋
- GitHub에 푸시

## 방법 2: 수동으로 명령어 실행

finance_dashboard 폴더에서 PowerShell을 열고 다음 명령어를 순서대로 실행:

### 1단계: Git 초기화 (처음 한 번만)

```powershell
git init
git branch -M main
```

### 2단계: 원격 저장소 연결 (처음 한 번만)

GitHub에서 저장소를 이미 생성했다면:

```powershell
git remote add origin https://github.com/kisshoi86/FINANCE_DASHBOARD.git
```

저장소가 이미 연결되어 있다면 이 단계는 건너뛰세요.

### 3단계: 파일 추가 및 커밋

```powershell
git add .
git commit -m "Initial commit: Finance Dashboard"
```

### 4단계: GitHub에 푸시

```powershell
git push -u origin main
```

## 인증 문제 해결

푸시할 때 인증 오류가 발생하면:

1. **Personal Access Token 생성:**
   - GitHub 웹사이트 접속
   - Settings > Developer settings > Personal access tokens > Tokens (classic)
   - "Generate new token" 클릭
   - `repo` 권한 체크
   - 토큰 생성 후 복사 (한 번만 보여줌!)

2. **푸시 시 사용:**
   - Username: GitHub 사용자 이름 (kisshoi86)
   - Password: 생성한 Personal Access Token

## 현재 상태 확인

푸시 전에 현재 상태를 확인하려면:

```powershell
# Git 상태 확인
git status

# 원격 저장소 확인
git remote -v

# 브랜치 확인
git branch
```

## 문제 해결

### "fatal: not a git repository"
→ finance_dashboard 폴더에서 실행 중인지 확인하고, `git init` 실행

### "remote origin already exists"
→ 원격 저장소가 이미 연결되어 있습니다. `git remote -v`로 확인

### "fatal: Authentication failed"
→ Personal Access Token을 사용하세요 (위의 "인증 문제 해결" 참고)

### "fatal: couldn't find remote ref main"
→ `git branch -M main`으로 브랜치 이름 확인
